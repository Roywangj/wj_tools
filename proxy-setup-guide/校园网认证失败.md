# Ubuntu 22.04 校园网认证脚本连接失败排查与解决

## 一、问题背景

在 Ubuntu 22.04 中运行校园网登录脚本 `wjlogin.py` 时出现连接卡住的问题：

```bash
python3 wjlogin.py
```

程序一直停在：

```
urllib.request.urlopen("http://10.0.0.55/cgi-bin/get_challenge?...")

KeyboardInterrupt
```

说明程序在访问 **校园网认证网关 `10.0.0.55`** 时发生阻塞。

---

# 二、问题原因

通过排查发现系统环境变量设置了 **HTTP 代理**：

```bash
http_proxy=http://10.106.1.36:7897
https_proxy=http://10.106.1.36:7897
all_proxy=socks5://10.106.1.36:7897
```

因此访问 `10.0.0.55` 时，系统尝试：

```
10.0.0.55 → 代理服务器 10.106.1.36:7897
```

而 **校园网网关是内网地址**，代理服务器无法访问，导致：

```
Connection timed out
```

Python 的 `urllib` 也会自动读取这些代理环境变量，所以脚本同样被代理拦截。

---

# 三、排查过程

## 1 查看路由

```bash
ip r
```

示例：

```
default via 10.106.14.1 dev ens15f0
src 10.106.15.178
```

说明：

* DHCP 正常
* 默认网关正常
* 网卡连接正常

因此 **网络本身没有问题**。

---

## 2 测试访问认证网关

```bash
curl -m 3 -v http://10.0.0.55/
```

返回：

```
HTTP/1.1 302 Moved Temporarily
Location: http://10.0.0.55/index_1.html
SRun portal server
```

说明：

✅ 成功连接校园网认证服务器。

---

# 四、解决方案

## 方法一：为内网地址禁用代理（推荐）

```bash
export NO_PROXY="10.0.0.55,10.0.0.0/8,10.106.0.0/16,localhost,127.0.0.1"
export no_proxy="$NO_PROXY"
```

再次测试：

```bash
curl -m 3 -v http://10.0.0.55/
```

应该看到：

```
Trying 10.0.0.55:80
Connected to 10.0.0.55
```

---

## 方法二：临时关闭代理

```bash
unset http_proxy https_proxy all_proxy
```

---

## 方法三：curl 单次绕过代理

```bash
curl --noproxy '*' http://10.0.0.55/
```

# 七、推荐优化

## 1 永久设置 NO_PROXY

在 `~/.bashrc` 或 `~/.zshrc` 添加：

```bash
export NO_PROXY="10.0.0.0/8,10.106.0.0/16,127.0.0.1,localhost"
export no_proxy="$NO_PROXY"
```

生效：

```bash
source ~/.bashrc
```

这样：

* 校园网地址不走代理
* 其他流量仍然可以使用代理

---

## 2 Python 脚本禁用代理（推荐）

在脚本开头加入：

```python
import urllib.request

opener = urllib.request.build_opener(
    urllib.request.ProxyHandler({})
)

urllib.request.install_opener(opener)
```

这样脚本访问网络时 **不会使用系统代理**。

---

## 3 为请求增加超时

避免网络异常导致程序卡死：

```python
urllib.request.urlopen(url, timeout=3)
```

---

## 4 避免在脚本中写明文密码

使用环境变量：

```bash
export SRUN_USER="你的账号"
export SRUN_PASS="你的密码"
```

Python：

```python
import os

username = os.environ["SRUN_USER"]
password = os.environ["SRUN_PASS"]
```

---

# 八、最终结论

问题根因：

```
系统代理拦截了访问校园网认证服务器（10.0.0.55）的请求
```

解决方法：

```
为校园网内网地址设置 NO_PROXY
```

最终效果：

* 登录脚本正常运行
* 校园网认证成功
* 外网访问正常
