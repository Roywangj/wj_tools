# Ubuntu 16.04 / 18.04 SSH 密钥登录配置

这份文档用于整理服务器从密码登录切换到 SSH 密钥登录的流程，适用于 `Ubuntu 16.04 / 18.04` 场景。

仓库里已经按角色拆分成两套说明：

- 普通用户：生成密钥、安装公钥、保存私钥、测试登录
- 管理员用户：在普通用户流程基础上，额外开启 SSH 公钥认证并关闭密码登录

## 推荐阅读顺序

### 普通用户

1. 在服务器上执行 `ssh-keygen` 生成密钥对
2. 把公钥追加到 `~/.ssh/authorized_keys`
3. 设置 `~/.ssh` 和 `authorized_keys` 权限
4. 保存私钥到本地
5. 在本地终端或 VSCode 中测试密钥登录

### 管理员用户

1. 先完成普通用户的密钥生成和登录验证
2. 编辑 `/etc/ssh/sshd_config` 开启 `PubkeyAuthentication`
3. 确认管理员账户可以通过密钥登录
4. 再将 `PasswordAuthentication` 设为 `no`
5. 重启 `sshd` 服务

## 常见注意事项

- 在保存私钥到本地之前，先重新打开终端，确保 `chmod` 修改已经生效。
- 如果在 Windows 上测试登录失败，优先检查私钥文件权限，而不是先怀疑服务器配置。
- 管理员不要在未验证密钥登录成功前直接关闭密码登录，否则容易把自己锁在服务器外。
- 如果你在多台服务器上都有账户，建议把不同服务器的私钥分目录保存，避免混用。
